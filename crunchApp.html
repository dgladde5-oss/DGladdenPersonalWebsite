Project Development Journey

Problem to be solved

Nov 23 2025

For a number of months I've noticed my parets often set alarms to remind them to book their gym classes. This current approach results is missed classes, rushing, and extreme fustration for both of them, so i stepped in to offer a solution. I decided the easiest way to overcome this challenge would be to have something to keep track of their planned classes, but that also broat with it the issue of them having to set their calenders. So finally i figured it out what I wanted to do, and built a system where all they needed was to list what they wanted. Enter the desired class into a document, and the system handles the rest.

Building

Dec 16, 2025

I have finally staryed building the program, and with it being almost 100% backend work, I'm finding it very difficult. The first step is to make sure this idea even works, and the first step to that is learning to automate. I found Selenium and Chromium two very powerful and free tools for automation. Using these tools, I split the work into 4 files; the driver, the configuration, the login function, the class booking. The first file created was the login finction since at this point I was there to monitor the outcome in real time. After starting the driver, I would have it navigate directly to the login page via a hardcoded url, this is certainly not the best choice, but this is a simple gift to close friends and family. Afterwards the driver would look the html for classes indicating the username and password fields and send the keys coded into the file. This step, after much struggle, succeded allowing me to move onto to booking the classes themselves.

Sprint 3

Dec 18, 2025

The next step was to make sure the program could navigate from the login screen to the form where classes are reserved and processed. Fundementally it was similar to the first step so this was not too much harder, it added more elements to jump across. After ensuring the project worked within this limited scope, the next step was to make it run automatically. This step was challenging as I have never done something quite like this before, but with lots of time and resources, even this was possible. I discovered the existence of a scheduler and this fit my needs perfectly, as it allowed the program to run continuously so long as the local device was on and active. I set the scheduler to run every hour ensuring they would never miss another class again, and with this being my first real project I was quite proud to reveal it as a Christmas day present. Over the course of this project I learned a lot about how the backend functions and how its acually quite powerful, being able to manipulate the front the way it does. Although this experience has also taught me that I need to practice a lot more, so I can continue to improve.
