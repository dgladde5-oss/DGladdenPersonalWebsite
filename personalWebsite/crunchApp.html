Project Development Journey

Sprint 1

Feb 4, 2026

This week began the first spring of the Photo App, an application template we recieved as part of my teams most recent academic assignment. While I am more familiar with front end than back end, I agree that a developer should be well rounded across the entire process, and be capable of performing any roll with competence. My task was to convert the current [userDetail] component to have the populating information be retrieved and uploaded into the preset React template. I will admit this was much more difficult than I assumed it would be; first I added a constructor to represent a user before having its data populated, next was a series of checks to determine if the component itself sucessfully loaded and updated the current users information, lastly was the render function to take this updated data and project it into the html file using the provided React template. Overall this was a good challenge and took me quite some time to figure out, but I feel I have earned a lot from the experience and hope to improve further.

Sprint 2

Feb 12, 2026

This second sprint was my first experience with MongoDB. Innitialy I assumed it would be similar to SQL which I have some experience using, but I was completely wrong. I was surprised to find these databases were so different, so much in fact, that very same day I began watching tutorials and reading notes on those differences and how to even make use of Mongo. Simple setup to my and many of my teamates devices was difficult to say the least, with constant bugs, operating systems, and differences in our respective machines causing many issues with coordination. After clearing these challenges were able to actually complete the assignment as intended. I was responsible for removing the servers depencency on the model and route requests to Mongo. Since everything else about Mongo was setup beforehand, my task was not nearly as difficult as the first. I removed the lines calling the model and replaced it with a Mongo query requesting the [userDetails] mentioned earlier.

Sprint 3

March 07, 2026

In this third sprint I was responsible for creating the login function for users, once again making use of MongoDB in order to retrieve user information like usermane and passwords. First I had to locate the schemas being used to control the shape of what a "user" is, after that, within the login function I called that schema and checked the input credentials against those already stored in the database. After validation the function either returned the correct information or an appropriate error message. This step was fairly straightforward, especially compared to what others on my team were assigned as their tasks.

Sprint 4 - Final

March 28, 2026

For this final sprint I was in charge of adding whatever single feature I deemed myself capable of adding, and I chose to add session persistance. I wanted the application to keep users logged in, even if the browser refreshed as I believe this is a very useful feature of modern browsing. This was actually far simplar than I thought, since the session already existed in the file, all I needed to do was check and see if it still esisted for the logged in user. After that it was as simple as just calling this function when ever the page loads itself. Over the course of the sememster I had the opertunity to work with some amazing and insightful people. Together we planned a mock software launch of our Photo App, giving a demonstration to peers and professors. I learned a lot about what goes on in the backend, and i truly hope to learn much more in the future.
